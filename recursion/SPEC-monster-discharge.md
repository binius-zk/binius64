## 0. Statement

Fix one CS shape S with term table T = the flattened list of N tuples τ_t = (x_t, y_t, s_t, op_t, m_t) — one per `ShiftedValueIndex` occurrence in slot m of AND-constraint x (source structs: crates/core/src/constraint_system.rs:134-143, :325, :332; the target leaves for this instantiation are AND-only, T_mul = ∅, so `lane` is hardwired; enforcement of AND-only is now explicit, §2 Phase 0 / §4 — [REV per S5]; the generic two-lane variant is §8.4). The deferred claim of leaf ℓ is exactly

  v_ℓ = Σ_{t<N} Λ^{(ℓ)}(m_t) · H^{(ℓ)}(op_t) · eq6(s_t, r_s^{(ℓ)}) · eq(x_t, r_x'^{(ℓ)}) · eq(y_t, r_y^{(ℓ)})

where the claim point c_ℓ = [r_zhat'(1), λ_and(1), λ_int(1), r_x'_and(23), r_x'_mul(0), r_j(6), r_s(6), r_y(23)] = 61 scalars for the nominal shape (order fixed at crates/verifier/src/protocols/shift/verify.rs:244-252), Λ^{(ℓ)}(m) = λ_and^{m+1}, and H^{(ℓ)}(op) = h_op from `evaluate_h_op` (crates/verifier/src/protocols/shift/monster.rs:27-90) — both transparent O(few hundred) mults from c_ℓ. [REV per I4b] `evaluate_h_op` consumes l_tilde = `lagrange_evals_scalars(subspace, r_zhat')` (verify.rs:270), whose 64 Lagrange evaluations require field inversions — free natively, and in-circuit realized as prover-supplied inverses + assert-mul pairs (~64 constraints/claim); accounted in §5/open questions.

[REV per S1 — headline invariant] The object this protocol certifies is not "a value v_ℓ" in the abstract: it is THE transcript element that leaf ℓ's `check_eval` consumes. `check_eval` asserts `witness_eval · monster_eval == eval` (verify.rs:315-316) where `witness_eval` is prover-recv'd and NOT independently PCS-bound (comment verify.rs:310-314); the equation pins `witness_eval` only because `monster_eval` is fixed. Once substituted, the discharge is the sole thing fixing it, so the discharged element and the multiplied element MUST be the same single recv'd Elem (§2 Phase 0, §5.1). Any implementation with two recv's of v_ℓ is forgeable regardless of everything below.

Given a commitment C_T to per-shape table data (STEP 2) and K same-shape claims {(c_ℓ, v_ℓ)}, prove all K with ONE argument. Verifier: polylog mults + K·O(|point|) eq work + (STEP 1: one native O(N) table pass) / (STEP 2: two BaseFold openings) + structural asserts (exactly-once substitution, coverage, AND-only, digest equality — [REV]). Prover: O(N) committed data per batch; O(K·N) uncommitted streaming mults (§8.1).

[REV per S2/I1 — trust roots, stated up front] Two items are trust-root assumptions, NOT proven in-protocol, with justification in §2/§5/§8: (T1) vk_digest ↔ cs_digest correspondence is established by deterministic regeneration audit (STEP 2 only; §8.12); (T0) scope is ONE flat aggregation level — the outer wrap's own IronSpartan wiring evaluations are NOT discharged by this protocol (they use a different seam and kernel; §5.3, §8.11).

## 1. Algebraic decomposition (all identities exact; char-2 field B128 GHASH)

### 1.1 [REV per I3 — single layout convention, used everywhere below]
All point vectors are written low-coordinate-first (little-endian, matching `eq_ind_partial_eval_scalars`, crates/math/src/multilinear/eq.rs:225-241: first coordinate = lowest index bit). All block/selector coordinates occupy the HIGHEST coordinates of their domain. Concretely:
- M_D (25 vars): index = a + 2^23·(c0 + 2c1); address a ∈ B_23 at coords [0..23), selector c = (c0,c1) at coords [23..25). Block map: c=(0,0)→D_x, (1,0)→D_y, (0,1)→D_g (zero-padded 2^11→2^23), (1,1)→zero.
- M_VK (27 vars): index = t + 2^25·(c0 + 2c1); t at coords [0..25), c at [25..27). Blocks: X, Y, U, 0.
- Phase-C union domain (27 vars): 25 low coords = t (blocks 00/01/10) or M_D's (a,c) (block 11); blk at coords [25..27). This matches `FracAddCheckProver`'s top-variable split (ip-prover/src/fracaddcheck.rs:~69-76): the four blocks merge in the layers nearest the root.
Layout is caller-defined (commit_interleaved treats the message as flat, crates/iop-prover/src/fri/commit.rs:39-91). The v1 spec mixed selector-low points ("M̃_D(00∥r_x')") with selector-high Phase-C formulas; v2 is selector-high everywhere, and all points below are rewritten accordingly.

### 1.2 Virtual columns and identities (unchanged content, points restated in §1.1 convention)
Define three per-row VIRTUAL columns for claim ℓ (never committed):
- E_x^{(ℓ)}[t] = eq(x_t, r_x'^{(ℓ)}), E_y^{(ℓ)}[t] = eq(y_t, r_y^{(ℓ)})
- E_g^{(ℓ)}[t] = G^{(ℓ)}(u_t), u_t = 11-bit meta index (s_t bits 0-5, op_t bits 6-8, m_t bits 9-10), G^{(ℓ)}(u) = Λ^{(ℓ)}(m)·H^{(ℓ)}(op)·eq6(s, r_s^{(ℓ)}).

Then v_ℓ = Σ_t E_x^{(ℓ)}[t]·E_y^{(ℓ)}[t]·E_g^{(ℓ)}[t] (identical to `evaluate_matrices` + `evaluate_monster_multilinear_for_operation`, monster.rs:123-219, re-associated; λ^{m+1} slot weights confirmed at monster.rs:137-140, scatter at :197-198).

**(D) Marginalization identity.** For any ρ ∈ F^25, with D_x(a;ρ) := Σ_t eq(t,ρ)·⟦x_t=a⟧, D_y likewise, D_g(u;ρ) := Σ_t eq(t,ρ)·⟦u_t=u⟧: Ẽ_x^{(ℓ)}(ρ) = D̃_x(r_x'^{(ℓ)}), same for y, i.e. reduced column evals ARE histogram MLEs at the claim's own point.

**(G) Eight-point decomposition.** In char 2, λ^{m0} = (λ+1)·eq(m0, (λ+1)^{-1}+1), λ^{2m1} = (λ+1)²·eq(m1, ((λ+1)²)^{-1}+1) (requires λ_and ∉ {0,1}, FS-random, abort ≤ 2/2^128/claim). With H^{(ℓ)}(op) = Σ_{o∈[8]} h_o^{(ℓ)}·eq3(op, bits(o)):
  Ẽ_g^{(ℓ)}(ρ) = Σ_{o=0}^{7} γ_{ℓ,o}·D̃_g(z_{ℓ,o}), γ_{ℓ,o} = h_o^{(ℓ)}·λ(λ+1)³, z_{ℓ,o} = (r_s^{(ℓ)}(6) ∥ bits(o)(3) ∥ p_0, p_1) ∈ F^11.

### 1.3 [REV per S4/I2 — padding rule, parity-corrected]
Pad T from N to N_pad with (N_pad − N) copies of ONE fixed dummy tuple τ_d = (x_d=0, y_d=0, u_d=0) (s=0, op=0, m=0), appended after the canonical real rows. In char 2, an EVEN number of dummy copies cancels identically in every sum. Since N = Σ|operands| has no parity guarantee (unmeasured; open question 1), the protocol does NOT require evenness. Instead: parity := (N_pad − N) mod 2 is frozen into the VK/statement metadata, and Phase A runs on the corrected sums

  sums[ℓ] := v_ℓ + parity · w_d(c_ℓ),  w_d(c_ℓ) = λ_and · h_0^{(ℓ)} · Π_i(1+r_s,i^{(ℓ)}) · Π_i(1+r_x',i^{(ℓ)}) · Π_i(1+r_y,i^{(ℓ)})

— the single unpaired dummy row's weight, transparent (~60 mults/claim, reusing the h_op evaluation the verifier already performs; eq(0,r) = 1+r in char 2). This is EXACT: Σ over the full B_25 cube = v_ℓ (real rows) + (N_pad−N)·w_d = v_ℓ + parity·w_d. The correction touches ONLY Phase-A sums; identities (D), Phase B, and Phase C consistently range over all N_pad rows including pads on both sides (verified by both reviews: the pad is a hazard only in the unweighted sum, never in the marginals). Note sums[ℓ] is derived Elem-arithmetically from THE recv'd v_ℓ element (public-derived w_d added), preserving the §0 single-source invariant. Do NOT fix parity by touching the CS or duplicating a real row (self-cancels the real row). Do not dedup real duplicate ShiftedValueIndexes — commit the literal list.

## 2. Protocol (four phases, one transcript)

Committed objects. STEP 2 VK (once per shape): three SCALAR address columns over t ∈ B_25 — X[t] = ι(x_t)+κ_x, Y[t] = ι(y_t)+κ_y, U[t] = ι(u_t)+κ_u — laid out per §1.1 as M_VK = [X | Y | U | 0], one 27-var non-ZK oracle via `fri::commit_interleaved` (crates/iop-prover/src/fri/commit.rs:39-52; transcript-free, digest = vk_digest). [REV per S3 — tag construction] ι maps address bit k to basis element β_k, k ∈ [0,23), V := ι(B_23) = span(β_0..β_22). Tags are the linear extension to the NEXT TWO basis elements: κ_c := ι'(c) with ι'(c0,c1) = c0·β_23 + c1·β_24, i.e. κ_x = 0, κ_y = β_23, κ_u = β_24, κ_pad = β_23+β_24. Then for c ≠ c', κ_c + κ_{c'} = ι'(c⊕c') ∈ span(β_23,β_24)\{0}, which intersects V trivially — the four pole families are cosets of V that are pairwise DISJOINT BY CONSTRUCTION ("field-distinct" is insufficient: cosets of the same F2-subspace collide unless the tag DIFFERENCE lies outside V). VKGen asserts κ_i ⊕ κ_j ∉ V for all i ≠ j at build time regardless. A bonus: emb(a,c) = ι(a) + ι'(c) is the plain linear embedding of the 25-bit M_D index under the extended basis, and Σ_c eq(c,σ)·κ_c = ι̃'(σ) (MLE of a linear map is itself), simplifying the Phase-C transparent. Per batch (STEP 2 only): M_D per §1.1, a 25-var non-ZK oracle. Nothing here is secret; the discharge uses NO ZK masking anywhere (avoids the mask-reuse trap and the `log_batch_size == 1` restriction of the ZK batch path).

**Phase 0 — statement absorption and structural preconditions.** [REV per S1/S2/S5 — this phase is now a set of MUSTs, not prose.]
(P0.1) [REV S2] Observe into the transcript, BEFORE μ is sampled: the full VK metadata blob VKM = {vk_digest, cs_digest, lane=AND-only flag, N, N_pad, parity, dims (25/23/11), tag basis indices (23,24), FRIParams fields (log_inv_rate, n_test_queries, fold arities, strategy), Merkle hash-suite id, canonical row-order version} (STEP 2), or {cs_digest, N, N_pad, parity, K} (STEP 1). Rationale (hard, not hygiene): `basefold::verify` takes the commitment as an ARGUMENT and never absorbs it (crates/iop/src/basefold.rs:57-106), and FRI query indices are `transcript.sample_bits` (crates/iop/src/fri/verify.rs:200) — an unobserved vk_digest is not bound by the queries and admits table-swap. cs_digest := H(canonical `SerializeBytes` bytes of the prepared ConstraintSystem) — canonical versioned serialization exists in-repo (constraint_system.rs:607-616).
(P0.2) [REV S2] Native precondition at the final verifier: assert VKM.cs_digest == H(serialize(leaf_cs)) against the leaf CS fixed by the wrap statement (STEP 1 has the cs in hand for the D rebuild and checks directly; STEP 2 checks the statement's leaf_cs_digest == VKM.cs_digest). This makes a same-shape rewired table unswappable without changing the CS identity the wrap itself was built against. The residual vk_digest↔cs_digest content link is trust root T1 (§8.12), NOT a proof.
(P0.3) [REV S5] Admission assert (both at VKGen and at capture time): ANDONLY(cs) — every `MulConstraint` equals the empty-operand default produced by `validate_and_prepare` padding (constraint_system.rs:560-574; MIN_MUL_CONSTRAINTS=1, consts.rs:29). Then r_x'_mul has length 0 and intmul_part ≡ 0 (empty operands ⇒ zero terms in `evaluate_matrices`), so monster_eval = bitand_part = the T-sum exactly (verify.rs:289-306). A CS failing ANDONLY is REJECTED by this discharge (route to the two-lane variant, §8.4) — previously a silent completeness/soundness hole.
(P0.4) [REV S1] Claim intake. Integrated path (the wrap): the c_ℓ components are already transcript-determined (sample/observe-derived during the K in-circuit leaf verifications); each v_ℓ is bound by the ONE recv the substituting channel performs at the monster call site (§5.1) — it MUST NOT be recv'd or observed a second time (in the symbolic builder it cannot even be observed: it is recv-tagged). Coverage assert: after the K leaf verifies, the recorder sink holds exactly K claims, each with |inputs| equal to the shape's expected arity (61 nominal), and Phase A's sums vector is exactly the sink's v-Elems (parity-corrected per §1.3) in sink order — no captured claim may bypass the discharge, since an undischarged v_ℓ leaves that leaf's check_eval vacuous. Standalone path: observe all (c_ℓ, v_ℓ) explicitly (`IPVerifierChannel::observe_many`, crates/ip/src/channel.rs:78-80); here the (c,v) pairs ARE the statement and S1 does not arise.

**Phase A — one batched degree-3 sumcheck over t (K claims → shared point ρ).**
Verifier: `sumcheck::batch_verify(n_vars=25, degree=3, sums=&[sums[1..K]], channel)` (crates/ip/src/sumcheck/batch.rs:37-60) → (μ, e_A, challenges ρ). All sums constituents are FS-bound before μ: v_ℓ by recv during leaf verifies, parity/N by P0.1, w_d by c_ℓ. Prover: K instances of a VENDORED `CubicProductSumcheckProver` (product of three multilinears; ~120 LoC mirror of crates/ip-prover/src/sumcheck/bivariate_product.rs:19-27 implementing `SumcheckProver`, crates/ip-prover/src/sumcheck/common.rs:29-77) over E_x,E_y,E_g^{(ℓ)}, driven by `batch_prove_and_write_evals` (crates/ip-prover/src/sumcheck/batch.rs:126-141).
[REV per I4a — transcript layout conforms to the driver.] The driver writes each prover's FULL multilinear-eval list at finish (`SumcheckProver::finish` contract "evaluations of all multilinears", common.rs:74-76): for the cubic prover that is THREE evals per claim, (a_ℓ, b_ℓ, g_ℓ). Verifier flow: (i) recv the 3K finish evals in driver order; (ii) check Horner(μ; {a_ℓ·b_ℓ·g_ℓ}) == e_A (the intmul verify_phase_3 pattern: per-claim transparent recomputation + one assert_zero; crates/verifier/src/protocols/intmul/verify.rs:111-183); (iii) prover then sends the 8K values d_{ℓ,o} (claimed D̃_g(z_{ℓ,o})); verifier recv's them and checks per claim g_ℓ == Σ_o γ_{ℓ,o}·d_{ℓ,o} (γ from `evaluate_h_op`, transparent; includes the l_tilde Lagrange step, §0). Soundness unchanged: g_ℓ is bound by the batched sumcheck, each d by Phase B, and the (G) identity is exact algebra linking them. By (D)/(G), correctness of {a_ℓ, b_ℓ, d_{ℓ,o}} is now equivalent to 10K MLE-eval claims on M_D, at points written per §1.1 [REV per I3]:
  {M̃_D([r_x'^{(ℓ)}, 0, 0]) = a_ℓ}, {M̃_D([r_y^{(ℓ)}, 1, 0]) = b_ℓ}, {M̃_D([z_{ℓ,o}, 0^{12}, 0, 1]) = d_{ℓ,o}} — trailing coordinates are the selectors (c0,c1); the 0^{12} mid-segment plus c=(0,1) selects the zero-padded D_g block.

**Phase C (STEP 2 only) — well-formedness of M_D vs M_VK via ONE weighted fracaddcheck.**
Prover commits M_D (digest_D observed — MUST, same P0.1 rationale). Verifier samples τ. Claimed rational identity over the 27-var union domain (§1.1 layout; blk at the TOP two vars): blk00: Σ_t eq(t,ρ)/(τ+X[t]); blk01: Σ_t eq(t,ρ)/(τ+Y[t]); blk10: Σ_t eq(t,ρ)/(τ+U[t]); blk11: Σ_{a,c} M_D(a,c)/(τ+emb(a,c)), emb(a,c) = ι(a)+ι'(c) — total ≡ 0. With tags coset-disjoint [REV S3], partial-fraction uniqueness forces, pole by pole, D_x(a) = Σ_{t:X[t]=ι(a)+κ_x} eq(t,ρ) — the histogram definition; unmatched κ_u poles auto-zero the D_g pad region and unmatched κ_pad poles force the (1,1) block to 0; no RLC coefficient needed. This is the WEIGHTED (coefficient-matching) use of fractional sums: numerators are field eq-weights, not integer multiplicities — the char-2 even-multiplicity cancellation attack on LogUp-style counting does not apply.
Run `fracaddcheck::verify(k=27, FracAddEvalClaim{num_eval: 0, den_eval: d_root, point: vec![]}, channel)` (crates/ip/src/fracaddcheck.rs:26-84: root point empty, per layer `batch_verify_mle(&point,2,[num,den])` + 4 recv'd evals + cross-multiplied composite (NO division) + extrapolate_line, point grows one var/layer → leaf claims num̃(π), deñ(π) at one π ∈ F^27). ## 3. Soundness accounting (96-bit target, |F| = 2^128)

Challenge order: statement/VKM → [K leaf verifies: c_ℓ sampled, v_ℓ recv'd] → μ → ρ (A) → 3K finish evals → 8K d's → digest_D → τ → fracaddcheck layers → φ → σ → m → ρ_c → FRI. Every challenge is sampled after everything it must bind; recv observes into FS (`FiatShamirBuf::advance`, crates/transcript/src/fiat_shamir/buffer.rs:24-32). Error terms:
- Phase A: Horner-RLC (K−1)/2^128 + sumcheck 25·3/2^128. The g-decomposition and parity-correction checks are exact algebra [REV].
- Phase B: RLC ≤ (10K)/2^128 + sumcheck 25·2/2^128.
- Phase C: rational-identity SZ in τ: ≤ 2^27 simple poles, cleared-denominator degree < 2^28 ⇒ ≤ 2^-100 (dominant IP term). Layer sumchecks ≈ 27²·c/2^128. Honest abort on τ hitting a pole ≤ 2^27/2^128. d_root ≠ 0 asserted; no in-protocol division (cross-multiplied composites, fracaddcheck.rs:61-65).
- Corner trick: ≤ 2/2^128 (error vector fixed before ρ_c). λ_and ∈ {0,1} abort ≤ 2/2^128/claim.
- [REV S3] Tag-coset condition is BY CONSTRUCTION (ι' into β_23,β_24), asserted at VKGen — zero residual probability, vs 2^-105 for random tags and total failure for in-V structured tags.
- [REV S1/S2/S5] The substitution single-source, coverage, arity, VKM-observation, cs_digest, and ANDONLY checks are structural (probability-free); their absence is what the two majors exploit, so they are protocol steps, not implementation advice.
- PCS: BaseFold/FRI parameters pinned in VKM (P0.1) to the same ≥96-bit budget as leaf proofs.
Char-2 subtleties: eq(b,r)=1+b+r affine (eq.rs:174-183); even duplicates cancel (used constructively, §1.3, with parity correction for the odd case); multiplicity-LogUp not used; only verifier-local inversions ((λ+1)^{-1}, l_tilde). Total ≪ 2^-96 for K ≤ 2^20.

## 5. Integration (multi-inner wrapper) [REV per S1 and I1 — rewritten]

### 5.1 Capture/substitution seam (leaf monster) — single-source invariant
`compute_public_value`'s contract allows not invoking f (crates/ip/src/channel.rs:100-111). The vendored SubstitutingChannel wraps the shared wrapper channel during each inner leaf IOP verify and, at the monster call site (crates/verifier/src/protocols/shift/verify.rs:254 — the SOLE compute_public_value call in the leaf verifier, verified by grep this session; the other protocol site, spartan-verifier/src/lib.rs:207, belongs to the outer verifier and is never wrapped), does exactly: (1) assert |inputs| equals the shape's claim arity (61 nominal) and that inputs are public-tagged (wrapper impls debug-assert this, spartan-verifier/src/wrapper/builder_channel.rs:224-237); (2) v := inner.recv_one() — the ONE and ONLY recv/binding of v_ℓ (recv feeds FS, buffer.rs:24-32); (3) record ClaimRecord{inputs-Elems, v-Elem} to the sink; (4) return that same Elem. The returned Elem is what `check_eval` multiplies (verify.rs:315-316), and the RECORDED Elem — the same wire in symbolic execution, the same in-memory value at the same transcript offset in native execution — is what Phase A consumes as sums[ℓ] (after the §1.3 parity add). FORBIDDEN: any second recv or observe of v_ℓ, or constructing sums from a re-read of the transcript — that reopens the two-unknown equation `witness_eval·v = eval` (witness_eval is prover-recv'd and not independently bound, verify.rs:310-314) and is a clean forgery. Exactly-once + coverage asserts per P0.4. Tag hygiene: the substituted result is recv-tagged (non-public); its sole consumer is the :315-316 mul + assert_zero and it never feeds another compute_public_value's inputs, so the public-tag contract (channel.rs:94-99) is not violated.

### 5.2 Placement in the flat wrap
The discharge runs as one more inner protocol segment of the flat multi-inner wrap (recursion-window/src/multi_zk.rs pattern), AFTER the K leaf IOP verifications, on the same transcript. Phases A/B/C use only `IPVerifierChannel` ops (batch_verify, sumcheck::verify, fracaddcheck::verify, recv/sample/assert) ⇒ they symbolically execute through `IronSpartanBuilderChannel` into outer-circuit constraints — in-circuit cost ≈ 150-250K B128 mults at K=64 (round checks + 10K eq_inds + K h_op evaluations incl. l_tilde witness-inverses [REV I4b]), vs K×51.7M removed. The PCS openings and (STEP 1) the native D check cannot be circuit-ized (Merkle queries escape the channel abstraction; precedent crates/iop/src/basefold_zk_channel.rs:157-167) — they join the final verifier's native endgame alongside the wrap's own combined opening.

### 5.3 [REV per I1 — corrected scope: ONE aggregation level; the outer CS is NOT "discharged identically"]
v1 claimed a node's own outer CS monster claim "is deferred identically and discharged by the PARENT against T_outer". That misdescribed the outer proof system. The outer system is IronSpartan (crates/spartan-verifier), and its O(N_outer) sparse-wiring work uses DIFFERENT seams and a DIFFERENT kernel:
- Only the PUBLIC-segment wiring eval goes through `channel.compute_public_value` (spartan-verifier/src/lib.rs:197-218, calling `evaluate_wiring_mle_public`, wiring.rs:74-99) — and that closure itself scans ALL outer mul-constraints, i.e. O(N_outer) native work per artifact at the final verifier TODAY.
- The precommit/private segment wiring evals are `TransparentEvalFn` closures (`wiring::eval_transparent`, wiring.rs:19-34) handed to `channel.verify_oracle_relations` (lib.rs:226-253) and evaluated natively by the final verifier at the combined-opening point — kernel per term: eq(x_t,r_x)·eq(y_t,r_y)·λ^{m_t}, slots m ∈ {0,1,2} batched by `evaluate_univariate` (wiring.rs:41-70, lib.rs:185); no h_op, no s, no op dims.
Therefore: (i) THIS SPEC'S SCOPE IS ONE FLAT AGGREGATION LEVEL — K same-shape leaves → 1 combined proof, whose final verifier = outer IronSpartan verify (retaining its own ~4 O(N_outer)-scale native wiring evaluations: 3 segment transparents + the public closure — a pre-existing Phase-1b cost this spec neither adds to nor removes) + the discharge endgame (STEP 1 native pass / STEP 2 two openings). (ii) Multi-level trees are OUT OF SCOPE (§8.11): they require BOTH a second capture seam (a TransparentEvalFn/verify_oracle_relations recorder — absent from §8.6's vendored list in v1, now listed as Variant-B work) AND in-circuit verification of a child's native endgame (FRI/Merkle), which this spec explicitly does not attempt. (iii) The outer-wiring discharge itself (Variant B, §8.11) is a STRICT SIMPLIFICATION of the leaf protocol — table rows (x_t, y_t, m_t) per segment, weight Λ(m)=λ^m decomposing into 2 eq factors via the same char-2 identity, ONE M_D-style point per claim (no 8× blowup, no U/G column) — but it is a protocol VARIANT with its own VK, not a reuse, and it is UNBUILT. All v1 claims about L2/L3 nodes and a "root small-table native pass" are withdrawn from the deliverable and restated under §8.11.

## 8. LIMITATIONS

1. **Prover is O(K·N) in streaming mults.** Inherent: K independent FS-random weightings each touch N terms. K-independent: all committed data, Merkle/PCS work, proof size (up to ~176B·K), final verifier.
2. **Same-CS-shape per batch.** One VK per shape; mixed windows partition into per-shape batches (shared transcript; unequal sizes via PaddedSumcheckDecorator or separate instances).
3. **Fixed-CS-shape leaf surgery still required for production windows** (real leaf shapes vary with content, so pinning one shape S per VK needs leaf surgery).
4. **AND-only-lane hardwiring — now ENFORCED, not assumed [REV S5].** ANDONLY(cs) asserted at VKGen and capture (P0.3); non-conforming CS rejected. Generic shapes need the two-lane variant: second table/VK (or lane tag), slots (a,b,lo,hi) ↦ λ_int^{1..4} (shift/verify.rs:289-306), own r_x'_mul dims.
5. **N dimension risk [REV S4/I2: parity de-risked].** Real N ≈ 30-34M (inferred, unmeasured) — one pass over `cs.and_constraints` must fix N_pad ∈ {2^25, 2^26} before VK freeze; parity is handled by the §1.3 transparent correction and recorded in VKM, so N's parity no longer constrains anything.
6. **Vendored components** (prover-side/glue; zero verifier-side protocol novelty): CubicProductSumcheckProver; SubstitutingChannel with single-source/exactly-once/arity/coverage asserts [REV S1]; W_eq builder; native histogram evaluator; discharge orchestration; deterministic VKGen (tags/parity/ANDONLY/cs_digest [REV]); chunked Phase-C tree builder; genericized wrapped-channel to interpose the SubstitutingChannel. [REV I1] NOT vendored here: the TransparentEvalFn recorder (Variant B, §8.11).
7. **STEP 2 ops burden**: 2^27 VK oracle re-committed per prover process; VKM pins FRIParams/strategy/hash-suite (P0.1); two PCS openings (~200-300 KB) unless a non-ZK batched opening is added later.
8. **Phase A memory** 3·16·K·N bytes at round 1 → sub-batch knob G (separate ρ per sub-batch; each is its own discharge instance).
9. **8× g-claim blowup** from H's non-factorability — accepted (each g-point is 2^11-cheap prover-side).
10. **External review item**: weighted-fracaddcheck (coefficient-matching partial fractions) in char 2 vs LogUp/Habock literature before code freeze; fallback = tag-packed prodcheck multiset argument (costlier).
11. **[REV per I1] Variant B (outer-wiring discharge) — specified in outline, UNBUILT; multi-level trees out of scope.** Deferring the outer IronSpartan wiring evals needs: a recorder interposed at the TransparentEvalFn/verify_oracle_relations seam (lib.rs:226-253) plus capture of the public-closure value (lib.rs:197-218); a per-segment table of (x_t,y_t,m_t) rows with kernel eq·eq·λ^m, Λ(m)=λ^m (m∈{0,1,2}) decomposing into 2 eq factors (one derived point per claim; no U/G column, no 8× blowup); its own VK per outer shape (uniform once arity + leaf shape are fixed). Even with Variant B, tree levels ≥2 additionally require in-circuit verification of a child's native endgame (FRI/Merkle) or an accumulation scheme — a separate design. Until then: single-level K→1 aggregation, final verifier retains the wrap's ~4 O(N_outer)-scale native wiring evals (pre-existing Phase-1b cost).
12. **[REV per S2] Trust root T1 (STEP 2 only): vk_digest ↔ CS content is an audited, not proven, link.** In-protocol we get: vk_digest FS-bound (P0.1, else FRI queries don't bind the table — fri/verify.rs:200), and VKM.cs_digest == leaf CS identity (P0.2). The remaining link — that the committed X/Y/U columns are the canonical table of THAT CS — is discharged by deterministic regeneration: VKGen is a pure function of the canonical CS serialization (constraint_system.rs:607-616); any party recomputes and compares digests (tens of seconds); CI-enforced. Justified scope-out: the CS is the system's trust root (today's verifier natively rebuilds it), so no smaller root exists to reduce to; proving VKGen in zero knowledge would be a circuit for the generator, out of scope. STEP 1 has no T1 at all.
